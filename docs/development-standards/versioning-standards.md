# Versioning Strategy

ReliefRoot follows Semantic Versioning (SemVer).

Format:

MAJOR.MINOR.PATCH

Example:

1.4.2

---

## MAJOR

Increment when making incompatible changes.

Examples:

2.0.0

---

## MINOR

Increment when adding backwards-compatible functionality.

Examples:

1.5.0

---

## PATCH

Increment when fixing bugs.

Examples:

1.4.3

---

## Development Phases

Sprint 0

Project foundation

Version:

0.0.x

---

Sprint 1+

Active development

Version:

0.x.x

---

Minimum Viable Product

First public release

Version:

1.0.0

---

Future Releases

Continue following Semantic Versioning.

Examples:

1.1.0

1.2.0

1.2.5

2.0.0

---

## Git Tags

Every release should receive a Git tag.

Examples:

v0.1.0

v0.5.0

v1.0.0

---

## Release Checklist

- Tests pass
- Documentation updated
- Version updated
- Changelog completed
- Git tag created
- GitHub release published

---

## Principle

Version numbers communicate change.

Users should understand the impact of an update simply by reading the version number.
